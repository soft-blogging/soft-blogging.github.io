    Что такое Gminer?

    Gminer - это программное обеспечение для майнинга нескольких монет, разработанное DevelSoftware1. Эта программа для майнинга может добывать криптовалюты, такие как Ethereum, Beam, Grin, Zelcash, Aeternity, Ravencoin и другие. Gminer — это программа командной строки. Поддерживается двойной майнинг GMiner ETH + TONCOIN.

    Поддерживает ли Gminer AMD?

    Программа ориентирована на платформы AMD и NVIDIA. Софт поддерживает самые популярные алгоритмы, такие как: Ethash, ProgPoW, KAWPOW, Equihash, CuckooCycle.

    Какова комиссия разработчика?

    Алгоритм Комиссия
    eth, ethash 1%
    etc, etchash 1%
    kawpow, rvn, ravencoin 1%
    cortex 5%

    Комиссия разработчика2 составляет 1% для Ethereum и Ethereum Classic. Комиссия остальных монет - от 2% до 5%.

    Как настроить Gminer для дуал майнинга?

    Шаг 1 - Скачайте майнер

    Чтобы загрузить Gminer, нам нужно перейти на официальный GitHub. Там вы найдете файлы для скачивания. В самом верху страницы мы увидим последнюю версию для Windows, а также последнюю версию для Linux.

    Выберите правильный файл для вашей операционной системы. Если вы используете Windows, выберите ZIP-файл. Для Linux — файл с расширением .tar.xz.

    Извлеките загруженный файл. Для распаковки архива лучше использовать WinRAR.

    Шаг 2 - Настройте Gminer

    После извлечения архива вы увидите файлы с типом windows batch file.

    Вам нужно вписать в эти файлы ваши адреса монет для майнинга.

    1. Для дуал майнинга используйте файл mine_eth+ton.bat.
    2. Для майнинга одной монеты, например Ethereum, используйте файл Ethereum-ethermine.bat.

    В этом руководстве мы использовали настройку дуал майнинга Ethereum и TON на пуле Ethermine Pool. Вот шаги:

    1. Ищем в папке криптовалюту, которую хотим добывать. В нашем случае это Ethereum + TON.
    2. Ищем подходящий майнинг-пул: Ton Whalestonpool и Ethermine Pool.
    3. Щелкните правой кнопкой мыши файл .bat и откройте его с помощью Блокнота Windows.
    4. Появится строка параметров, в которой мы должны настроить различные команды.
    5. Первая — это сервер пула, к которому мы подключаемся. В нашем случае можно оставить eu1.ethermine.org:4444
    6. В команде --user мы меняем адрес кошелька Ethereum, который мы отметили зеленым цветом на следующем изображении.
    7. В команде --duser мы меняем адрес кошелька TON coin, который мы отметили красным цветом на следующем изображении.
    8. Сохраните конфигурацию и запустите файл, который вы отредактировали.

    Получите кошельки на бирже криптовалют.

    Ниже приведена настройка бат файла:

    miner.exe --algo ethash --server eth.2miners.com:2020 --user 0x5576999cc34c1c0030f28939444d6ba9cd6919c2 --dalgo ton --dserver wss://eu1.stratum.ton-pool.com/stratum --duser EQDk8olXYxBsrxiKvZSQMO1-Jjmx9_e-yNjvePJ3dxHlnAwf

    Итак, измените строку после --user и --duser.

    Вместо 0x5576999cc34c1c0030f28939444d6ba9cd6919c2 впишите адрес вашего Ethereum кошелька. Вместо EQDk8olXYxBsrxiKvZSQMO1-Jjmx9_e-yNjvePJ3dxHlnAwf впишите адрес TON.

    Нужно изменить только часть этого кода. Ничего больше менять не нужно:

    Дважды щелкните .bat файл. Стратует процесс майнинга.

    Вы увидите следующее окно cmd в Windows:

    Эта консоль показывает текущий статус майнинга. Чтобы остановить майнер, нужно просто закрыть это окно. Посмотрите видео, чтобы создать более продуктивную настройку.

    Команды

    Для начала пример. Для использования команд выберите либо длинную --algo или короткую -a. Обе использваоть не нужно.

    Команды для подключения к пулу

    • --algo или коротко -a - выбрать алгоритм добываемой монеты (пример: --algo ethash, --algo kawpow, -a cuckatoo32, -a beamhash).
    • --server или коротко -s - адрес пула майнинга (пример: -s eu1.ethermine.org , -s eth-eu.f2pool.com).
    • --port или коротко -n - порт пула (пример: -n 5057, -n 1445).
    • --user или коротко -u - логин пула или адрес кошелька, имя воркера может быть указано с точкой (пример: -u sRuJK1BmA758GbOn.worker, -u RLJUrTXTFZbaMdq6Xfz574ESWtduSokhZo.rig0).
    • --worker - имя воркера для Ethash strarum, для пулов, которые не поддерживают wallet.worker (пример: rig0) .
    • --pass или коротко -p - пароль воркера или пароль пула по умолчанию. Значение по умолчанию равно x (пример: MyAwesomePassword).
    • --ssl - включить/отключить безопасное соединение с пулом (0 - выключено или 1 - включено), должно поддерживаться пулом, значение по умолчанию равно 0 .
    • --proxy - адрес прокси-сервера SOCKS5 (пример: --proxy 31.7.232.178:1080).
    • --proto - укажите режим протокола stratum, возможные значения: proxy и stratum, полезные для майнинга Ethash, значение по умолчанию - “proxy” (пример: --proto stratum).

    Выбор видеокарты и устройств

    --list_devices - показать список устройств, доступных для майнинга.
    --devices или коротко -d - разделенный пробелом список устройств cuda, значение по умолчанию - все доступные устройства (пример: -d 1 3 5). --nvml - включить/отключить NVML (статистическую библиотеку для устройств CUDA) (0 - выкл. или 1 - вкл.), значение по умолчанию равно 1.

    --cuda - включить/отключить платформу CUDA (0 - выкл. или 1 - вкл.), значение по умолчанию равно 1.

    --opencl - включить/отключить платформу OpenCL (0 - выкл. или 1 - вкл.) , значение по умолчанию равно 1.

    Команды для разгона и power limit

    --mt - разделенный пробелами список номеров настроек памяти для каждого устройства (диапазон от 0 до 6, 0 - отключить настройки), поддерживаются только графические процессоры Nvidia с памятью GDDR5X и GDDR5, требуется запуск майнера с правами администратора (пример: --mt 1 3 5).

    --pl - разделенный пробелом список ограничений мощности для каждого устройства в процентах (диапазон от 0 до 100 для графических процессоров Nvidia и от -50 до 50 для графических процессоров AMD, 0 - игнорировать), поддерживается только Windows (пример: --pl 30 0 50).

    --cclock - разделенный пробелом список смещений тактовой частоты ядра (для графических процессоров Nvidia) или абсолютных тактовых частот ядра (для графических процессоров AMD) для каждого устройства в МГц (0 - игнорировать), поддерживается только Windows, требуется запуск майнера с правами администратора (пример: --cclock 100 0 -90).

    --mclock - разделенный пробелом список смещений тактовой частоты памяти (для графических процессоров Nvidia) или абсолютных тактовых частот памяти (для графических процессоров AMD) для каждого устройства в МГц (0 - игнорировать), поддерживается только Windows, требуется запуск майнера с правами администратора (пример: --mclock 100 0 -90).

    --cvddc - разделенный пробелом список смещений напряжения ядра в % (для графических процессоров Nvidia) или абсолютных напряжений ядра (для графических процессоров AMD) для каждого устройства в mV (0 - игнорировать), поддерживается только Windows, требуется запуск майнера с правами администратора (пример: --cvddc 900 0 1100).

    --lock_voltage - разделенный пробелом список точек заблокированного напряжения для каждого устройства в мВ (0 - игнорировать), поддерживаются только графические процессоры Windows и Nvidia. Требуется запуск майнера с правами администратора (пример: --lock_voltage 900 0 1000).

    --lock_cclock - разделенный пробелом список заблокированных тактовых точек ядра для каждого устройства в МГц (0 - игнорировать), поддерживаются только графические процессоры Nvidia. Требуется запуск майнера с правами администратора (пример: --lock_cclock 1200 0 1500).

    --p2state - включить / отключить состояние P2, поддерживаются только графические процессоры Windows и Nvidia. Требуется запуск майнера с правами администратора.

    --kernel или коротко -k - разделенный пробелом список номеров ядер для каждого устройства (0 - авто, 1-6 - номер ядра, в настоящее время поддерживается 6 ядер для Nvidia на Ethash/Etchash) (пример: -k 1 3 5).

    Интенсивность

    --intensity или коротко -i - разделенный пробелом список интенсивностей (1-100), значение по умолчанию равно 100 (пример: -i 90 90 90) .

    --share_check - включить/отключить проверку общего доступа к процессору для майнинга Ethash (0 - выкл. или 1 - вкл.), Etcash, KAWPOW и ProgPoW, значение по умолчанию равно 1.

    Управление вентилятором и температурой

    --fan - разделенный пробелом список скорости вращения вентилятора для каждого устройства в процентах (диапазон от 0 до 100, 0 - игнорировать), поддерживается только Windows (пример: 60 0 90).

    --tfan - разделенный пробелом список целевых температур для вентилятора (0 - игнорировать), поддерживается только Windows (пример: --tfan 65 0 70).

    --tfan_min - разделенный пробелом список минимальной скорости вентилятора (0 - игнорировать) для параметра --tfan, поддерживается только Windows (пример: --tfan_min 30 0 35).

    --templimit или коротко -t - разделенный пробелом список температурных ограничений, при достижении предела графический процессор прекращает майнинг до тех пор, пока он не остынет (пример: -t 85 80 75).

    --templimit_mem или коротко -tm - разделенный пробелом список ограничений температуры памяти, при достижении предела графический процессор прекращает майнинг до тех пор, пока он не остынет (пример: -tm 95 100 105).

    --tfan_max - разделенный пробелом список максимальной скорости вентилятора (0 - игнорировать) для параметра --tfan, поддерживается только Windows (пример: --tfan_max 90 0 80).

    Команды дуал майнинга

    --dserver - адрес сервера для дуал монеты. --dalgo - алгоритм для дуал майнинга.
    --dport- порт сервера для дуал монеты.
    --duser - имя пользователя для дуал монеты.
    --dpass - пароль для сервера дуал монеты.
    --dssl - включить/отключить ssl для дуал сервера.
    --dworker - дуал имя воркера для пулов, которые не поддерживают wallet.worker.
    -di или --dual_intensity - разделенный пробелом список интенсивностей для дуал майнинга [0 - 128].

    Логи, журнал ошибок

    • --logfile или коротко -l" - имя файла для сохранения журналов на диске, значение по умолчанию равно /usr/user/miner.log, Windows: c:\miner.log ).
    • --log_date - включить/отключить дату в каждом сообщении, значение по умолчанию равно 0 (0 - выкл. или 1 - вкл.).
    • --log_stratum - включить/отключить передачу данных с сервером, значение по умолчанию равно 0 (0 - выкл. или 1 - вкл.).
    • --log_newjob - включить/отключить информацию о новых заданиях, значение по умолчанию равно 1 (0 - выкл. или 1 - вкл.).
    • --color или коротко -c - включить/отключить вывод цвета для консоли, значение по умолчанию равно 1 (0 - выкл. или 1 - вкл.).

    Сторожевой таймер - watchdog

    --watchdog или коротко -w - включить/отключить watchdog, watchdog отслеживает основные процессы майнинга и перезапускает приложение в случае сбоя или потери соединения с пулами, значение по умолчанию равно 1 (0 - выкл. или 1 - вкл.).

    --watchdog_restart_delay - задержка перезапуска майнера для watchdog в секундах, значение по умолчанию равно 10 (пример: --watchdog_restart_delay 1).

    --watchdog_mode - действие сторожевого пса при завершении работы майнера (0 - перезапуск майнера, 1 - перезагрузка системы), значение по умолчанию равно 0 (пример: --watchdog_mode 1).

    --min_rig_speed - минимальная скорость майнинг фермы, майнер завершает работу, если средняя скорость падает ниже заданного значения.

    --report_interval - интервал отчета статистики в секундах, значение по умолчанию равно 30 (пример: --report_interval 5).

    --api - порт сервера телеметрии, позволяет удаленно отслеживать состояние майнера, откройте ссылку в вашем браузере http://localhost:port (пример: 10050, 20030).

    Конфигурационный файл

    --config - указать конфигурационный файл.

    --pers - строка персонализации для алгоритма equihash (пример: BgoldPoW, BitcoinZ, Safecoin).

    Статистика хешрейта и дохода

    --pec - включить/отключить калькулятор энергоэффективности. Калькулятор энергоэффективности отображение статистики энергоэффективности графического процессора, более высокая загрузка процессора. Значение по умолчанию - 1 (0 - выкл. или 1 - вкл.).

    --electricity_cost - передайте стоимость электроэнергии в долларах США за кВтч, майнер сообщит о потраченных долларах на майнинг.

    DAG файл

    --dag_limit - разделенный пробелом список ограничений размера Dag файла в мегабайтах, чтобы отключить ограничение, используйте 0, по умолчанию 0 (пример: --dag_limit 4096 4096 4096).

    --safe_dag - разделенный пробелом список режимов генерации DAG (0 - автоматический, 1 - быстрый режим, 2 - безопасный режим, в автоматическом режиме графические процессоры GTX - быстрый DAG и RTX - безопасный DAG), поддерживаются только графические процессоры Nvidia, по умолчанию 0 (пример: --safe_dag 2 1 0).

    --cache_dag - включить/отключить кэширование DAG файла для майнинга Ethash + Zilliqa или Nicehash(0 - выкл. или 1 - вкл.), значение по умолчанию равно 1.

    --dag_gen_limit - максимальное количество параллельных генераций DAG, 0 - ограничение отключения, значение по умолчанию равно 0 (пример: --dag_gen_limit 3).

    --dag_mode - разделенный пробелом список режимов файла Dag (0 - автоматический, 1 - одиночный, 2 - двойной), значение по умолчанию равно 0 (пример: --dag_mode 2 1 0).

    Разное

    --help или коротко -h - отображение доступных опций.

    --version или коротко -v - версия программы для вывода в консоль.

    --user_id - вывод идентификатора пользователя.

    --secure_dns - включить/отключить только DNS по запросам HTTPS.

    --maintenance_server - адрес пула для обслуживания (пример: eu.btgpool.pro , eu1.zhash.pro ).

    --maintenance_port - порт пула для обслуживания (пример: 5057, 1445).

    --maintenance_user - логин пула или адрес кошелька для обслуживания.

    --maintenance_pass - рабочий пароль или пароль пула по умолчанию для обслуживания.

    --maintenance_ssl - включить/отключить безопасное соединение с пулом (0 - выключено или 1 - включено) для обслуживания, должно поддерживаться пулом, значение по умолчанию равно 0 .

    --maintenance_proto - укажите режим протокола stratum для обслуживания, возможные значения: proxy и stratum, полезные для майнинга Ethash, значение по умолчанию - proxy (пример: stratum).

    --maintenance_worker - имя воркера для Ethash strarum для обслуживания, для пулов, которые не поддерживают wallet.worker (пример: rig0).

    --maintenance_fee - процент платы за обслуживание.

    --tor - включение/отключение сетевых подключений через Tor.

    Конкурс

    --contest_wallet - кошелек Ethash, параметр необходим для участия в конкурсе.

    LHR

    Что такое LHR в майнинге?

    Для успешного обхода защиты LHR необходимо предоставить майнеру права администратора. Для этого щелкните правой кнопкой мыши исполняемый файл в папке программы и перейдите в свойства (properties).

    Поставьте галочку напротив нужного пункта во вкладке «Совместимость» (compatibility).

    Затем запустите bat-файл от имени администратора.

    Если майнер не запускается, просто запустите командный файл.

    Для редактирвоания настроек откройте bat-файл в любом текстовом редакторе.

    miner --algo ethash --server eu1.ethermine.org:4444 --user 0x5576999cc34c1c0030f28939444d6ba9cd6919c2.rig
pause

    Поддерживаются только графические процессоры Nvidia.

    Команды для снятия блокировки LHR:

    • --lhr - разделенный пробелом список режимов LHR (0 - авто, 1 - включено, 2 - выключено).
    • --lhr_tune - разделенный пробелом список значений настройки LHR, означающий процент разблокировки видеокарты (0 - авто), значение по умолчанию равно 0 (пример: --lhr_tune 72 71 73).
    • --lhr_autotune - разделенный пробелом список автоматической настройки LHR, 0 - выкл., 1 - вкл. (пример: --lhr_autotune 1 0 1).
    • --lhr_autotune_step - размер шага автоматической настройки LHR, значение по умолчанию равно 0.1 (пример: --lhr_autotune_step 0.2).
    • --lhr_mode - разделенный пробелом список режимов LHR (0 - режим энергосбережения, 1 - режим максимальной производительности), значение по умолчанию равно 1 (пример: --lhr_mode 1 0 1).
